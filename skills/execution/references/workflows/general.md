# General Workflow

适用于：没有专门 workflow 匹配的任何类型任务的兜底流程。

## 上下文探索

执行前，主动探索：
- `practices/` 下是否有适用于当前任务的规范
- `practices/` 下是否有适用的规范或方法论
- 项目中是否有相关的历史实现可参考
- 是否有对应的外部 Skill 可加载（优先使用专业 Skill）

## 执行原则

- 每步完成后验证结果，确认符合预期再继续
- 遇到不确定的决策点，暂停向用户确认
- 优先复用已有资源，避免重复造轮子

## Subagent 调度

分配任务给 Subagent 时，Task prompt 中必须包含：
- 明确的任务目标和验收标准
- 相关 reference 文件路径（让 Subagent 读取）
- 任务约束条件和边界
- 期望的输出格式

## 交付验证

- 输出是否满足用户的原始需求
- 是否符合探索到的规范要求
- 交付给用户确认，根据反馈迭代
