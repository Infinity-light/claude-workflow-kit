# 密钥读取器

## 触发场景

- 项目需要调用外部 API（LLM、多模态、向量化、支付、鉴权等）
- 需要配置模型名称、Base URL 等服务参数
- 项目缺少 `.env` 文件或缺少所需密钥

## 操作流程

### 第一步：读取全局密钥

读取 `~/.claude/glo.env`（即 `C:\Users\WaterFish\.claude\glo.env`），解析所需的密钥键值对。

文件格式为 `KEY：VALUE`（中文冒号分隔），每行一个键值对。

### 第二步：迁移到项目本地

将所需的密钥条目写入**当前项目根目录**的 `.env` 文件：

- 已有 `.env` → 追加不重复的条目
- 没有 `.env` → 创建新文件
- 格式转换：中文冒号 `：` → 英文等号 `=`，横杠 `-` → 下划线 `_`
- 示例：`DEEPSEEK-API-KEY：sk-xxxx` → `DEEPSEEK_API_KEY=sk-xxxx`
- 确保 `.gitignore` 包含 `.env`，防止密钥泄露

### 第三步：验证模型配置

当密钥涉及模型服务时，**必须先查询 API 确认可用模型，绝不凭记忆编造模型名称**：

1. 从 `.env` 中取得 `BASE_URL` 和 `API_KEY`
2. 调用 `GET {BASE_URL}/models` 查询可用模型列表
3. 根据返回结果选择正确的模型名称写入配置

## 注意事项

- 密钥内容**绝不**输出给用户，不在对话中展示完整密钥
- 代码中通过 `os.getenv()` 或对应框架的环境变量方式读取本地 `.env`
- 如果 `glo.env` 中没有所需的密钥，主动询问用户补充
